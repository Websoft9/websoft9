# Websoft9 手动测试指南

> 📖 **面向人群：** 非专业测试人员、社区贡献者  
> 🎯 **目标：** 在发布新版本前，确保所有核心功能正常工作  
> ⏱️ **预计耗时：** 45-60 分钟（全新安装）/ 20-30 分钟（升级测试）

---

## 🚀 快速开始

### 准备工作

1. **准备一台测试服务器**（建议 Ubuntu 24.04 或 CentOS Stream 9，最低 2C4G）
2. **安装 Websoft9**
3. **打开浏览器**访问 `http://服务器IP:9000`
4. **打开测试清单 Issue**，对照逐项测试

### 安装命令

```bash
# 全新安装
wget -O install.sh https://artifact.websoft9.com/websoft9/release/platform/install.sh && sudo bash install.sh

# 带参数安装（自定义端口和版本）
wget -O install.sh https://artifact.websoft9.com/websoft9/release/platform/install.sh && sudo bash install.sh --console-port 9000 --channel release --version "latest"

# 升级已有安装
wget -O install.sh https://artifact.websoft9.com/websoft9/release/platform/install.sh && sudo bash install.sh --version "latest"
```

## 🖼️ 截图指南

### 如何截图

- **Windows：** `Win + Shift + S`（截图工具）或 `PrtSc` 键
- **Mac：** `Cmd + Shift + 4`（选区截图）
- **浏览器截图：** Chrome/Edge 按 `F12` → `Ctrl+Shift+P` → 输入 "screenshot"

### 什么情况需要截图

- ✅ 功能正常时（作为通过证据，特别是不常见的功能）
- ❌ 出现错误提示或异常界面
- 🐛 操作结果与预期不符
- 📊 数据展示异常或空状态

## 🔧 测试技巧

### 容器状态检查
```bash
# 查看容器运行状态
docker ps -a --format "table {{.Names}}\t{{.Status}}"

# 查看容器日志
docker logs websoft9-apphub --tail 50

# 进入容器排查
docker exec -it websoft9-apphub bash
```

### 应用安装失败排查
1. 检查磁盘空间：`df -h`
2. 检查 Docker 状态：`systemctl status docker`
3. 查看安装日志：在「我的应用」中点击安装日志弹窗
4. 手动拉取镜像测试：`docker pull <镜像名>`

### SSL 证书问题排查
1. 确认域名已解析：`nslookup your-domain.com`
2. 确认 80 端口开放：`curl -I http://your-domain.com`
3. 查看 NPM 日志：`docker logs websoft9-proxy`
