功能测试矩阵

每个测试用户对应三列：结果 / 问题描述 / 截图或证据。当前用户来自新平台的产品认证用户列表。

建议的真实测试方式

  1. 测试负责人先统一创建一份发布前测试表，功能项固定，不允许每个人各写一套。
  2. 每个测试用户只改自己的那一组列，不覆盖其他人的结果。
  3. 发现问题时，直接在“问题描述”里记录现象、入口、复现步骤；截图列放截图文件名或工单链接。
  4. 最终按“P0 不通过 / 阻塞”数量决定是否允许发布。
  • 当前演示包含登录、应用部署、Compose、My Apps、Access、文件、设置等关键路径。
  • 已禁用或已删除的用户仍会显示，便于你判断是否应纳入发布前测试名单。
